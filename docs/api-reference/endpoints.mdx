## Overview

TrustLists provides RESTful API endpoints to access trust center data programmatically.

## Base URL

```
https://trustlists.org/api
```

## Endpoints

### Trust Centers

<ExpandableCard title="GET /trust-centers" icon="database">

Retrieve all trust centers or filter by platform.

**Parameters:**
- `platform` (optional): Filter by hosting platform (e.g., "Vanta", "Drata", "Secureframe")
- `limit` (optional): Number of results to return (default: 50, max: 100)
- `offset` (optional): Number of results to skip (default: 0)

<RequestExample>
```bash cURL
curl "https://trustlists.org/api/trust-centers?platform=Vanta&limit=10"
```
</RequestExample>

<ResponseExample>
```json Success
{
  "trustCenters": [
    {
      "name": "1Password",
      "url": "https://trust.1password.com",
      "platform": "Self-hosted",
      "description": "Password manager security and compliance information"
    },
    {
      "name": "Figma",
      "url": "https://www.figma.com/security/",
      "platform": "Self-hosted",
      "description": "Design tool security and compliance"
    }
  ],
  "total": 348,
  "limit": 10,
  "offset": 0
}
```
</ResponseExample>

<Tip>
Use the `platform` parameter to filter by hosting platform. TrustLists automatically detects platforms like Vanta, Drata, and Secureframe.
</Tip>

</ExpandableCard>

<ExpandableCard title="GET /stats" icon="chart">

Get statistics about the trust center directory.

<RequestExample>
```bash cURL
curl "https://trustlists.org/api/stats"
```
</RequestExample>

<ResponseExample>
```json Success
{
  "totalTrustCenters": 348,
  "platforms": {
    "Vanta": 45,
    "Drata": 32,
    "Secureframe": 28,
    "Oneleet": 15,
    "HyperComply": 12,
    "Self-hosted": 156,
    "Other": 60
  },
  "lastUpdated": "2024-01-15T10:30:00Z",
  "discoveryStats": {
    "lastDiscoveryRun": "2024-01-15T09:00:00Z",
    "newThisWeek": 8,
    "highConfidencePending": 3
  }
}
```
</ResponseExample>

<Info>
The `discoveryStats` field shows automated discovery metrics, including new trust centers found this week and high-confidence entries pending review.
</Info>

</ExpandableCard>

## Rate Limits

- **Free tier**: 100 requests per hour
- **No authentication required** for public endpoints

## Error Handling

All endpoints return standard HTTP status codes:

- `200` - Success
- `400` - Bad Request
- `404` - Not Found
- `429` - Rate Limit Exceeded
- `500` - Internal Server Error

<Warning>
Rate limits are enforced per IP address. If you exceed 100 requests per hour, you'll receive a 429 status code.
</Warning>

### Error Response Examples

<CodeGroup>
```json Bad Request (400)
{
  "error": "Invalid platform parameter",
  "message": "Platform must be one of: Vanta, Drata, Secureframe, Oneleet, HyperComply, Self-hosted, Other"
}
```

```json Rate Limit Exceeded (429)
{
  "error": "Rate limit exceeded",
  "message": "You have exceeded the rate limit of 100 requests per hour. Please try again later.",
  "retryAfter": 3600
}
```

```json Not Found (404)
{
  "error": "Not found",
  "message": "No trust centers found for the specified criteria"
}
```
</CodeGroup>
