## Overview

TrustLists provides RESTful API endpoints to access trust center data programmatically.

<Warning>
**Static API Limitation:** These are static JSON endpoints served by GitHub Pages. Query parameters like `search` do not filter results server-side. All filtering must be implemented client-side in your application.
</Warning>

## Base URL

```
https://trustlists.org/api
```

## Endpoints

### Trust Centers

<ExpandableCard title="GET /trust-centers" icon="database">

Retrieve all trust centers from the directory.

**Parameters:**
- None (returns all trust centers)

<RequestExample>
```bash cURL
curl "https://trustlists.org/api/trust-centers.json"
```
</RequestExample>

<ResponseExample>
```json Success
{
  "data": [
    {
      "name": "1Password",
      "website": "https://1password.com/",
      "trustCenter": "https://app.conveyor.com/profile/1password",
      "platform": "Conveyor",
      "iconUrl": "https://www.google.com/s2/favicons?domain=1password.com&sz=128"
    },
    {
      "name": "Figma",
      "website": "https://figma.com/",
      "trustCenter": "https://compliance.figma.com/",
      "platform": "Conveyor",
      "iconUrl": "https://www.google.com/s2/favicons?domain=figma.com&sz=128"
    }
  ],
  "meta": {
    "total": 441,
    "generated": "2024-01-15T10:30:00Z",
    "version": "1.0.0"
  }
}
```
</ResponseExample>

<Tip>
Use the `search` parameter to filter by company name. TrustLists automatically detects hosting platforms like SafeBase, Conveyor, and Vanta.
</Tip>

</ExpandableCard>

### Stats

<ExpandableCard title="GET /stats" icon="chart">

Get statistics about the trust center directory.

<RequestExample>
```bash cURL
curl "https://trustlists.org/api/stats"
```
</RequestExample>

<ResponseExample>
```json Success
{
  "totalCompanies": 441,
  "generated": "2024-01-15T10:30:00Z"
}
```
</ResponseExample>

<Info>
The stats endpoint provides basic directory statistics. Platform-specific counts are available through the main trust-centers endpoint.
</Info>

</ExpandableCard>

## Rate Limits

- **No explicit rate limits** - API is designed for reasonable usage
- **No authentication required** for public endpoints

## Error Handling

All endpoints return standard HTTP status codes:

- `200` - Success
- `400` - Bad Request
- `404` - Not Found
- `500` - Internal Server Error

<Warning>
API endpoints are designed for reasonable usage. Excessive requests may be throttled by the hosting provider.
</Warning>

### Error Response Examples

<CodeGroup>
```json Bad Request (400)
{
  "error": "Invalid search parameter",
  "message": "Search query must be a valid string"
}
```

```json Not Found (404)
{
  "error": "Not found",
  "message": "No trust centers found for the specified criteria"
}
```

```json Internal Server Error (500)
{
  "error": "Internal server error",
  "message": "Failed to fetch trust centers"
}
```
</CodeGroup>
